## Email Campaign

```yaml
type: campaign
name: My Email Campaign
campaign_type: email
description: Campaign description (optional)
workspace: Marketing Team

audience: ref:parent_segment_name
segment: ref:Segment Name

email:
  template: ref:Template_Name        # Existing template name (from tdx engage templates)
  subject: "Hello {{profile.first_name}}, check this out!"
  html_file: my-campaign.html        # HTML file in same directory as YAML
  variables:                         # Template variables (optional)
    - name: first_name
      display_name: First Name
      preview_value: John
      default_value: Customer

connector:
  email_sender_id: 0199xxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx  # From workspace email senders
  all_columns: false
  columns:
    - column: email
      source: email

utm:
  enabled: true
  medium: email
  source: treasuredata
  source_platform: treasuredata
  campaign: my-campaign-slug
  marketing_tactic: promo-type

notifications:
  events:
    - onSuccess
    - onFailure
  recipients:
    - 230906           # TD user IDs
```

## Push Campaign

```yaml
type: campaign
name: My Push Campaign
campaign_type: push
description: Push notification campaign
workspace: Marketing Team

audience: ref:parent_segment_name
segment: ref:Segment Name

push:
  title: "Push notification title"
  message: "Push notification body text"
  push_type: notification             # notification
  on_click_behavior: open_deeplink    # open_deeplink (optional)
  image_url: https://example.com/image.png  # (optional)

connector:
  all_columns: false
  columns:
    - column: age
      source: age

utm:
  enabled: true
  medium: push
  source: treasuredata
  source_platform: treasuredata
  campaign: my-push-campaign
  marketing_tactic: mobile-push-followup
```

## Fields Reference

### Top-level

| Field | Required | Description |
|-------|----------|-------------|
| `type` | Yes | Always `campaign` |
| `name` | Yes | Campaign display name |
| `campaign_type` | Yes | `email` or `push` |
| `description` | No | Campaign description |
| `workspace` | Yes | Engage workspace name |
| `audience` | Yes | Parent segment: `ref:<name>` |
| `segment` | Yes | Child segment: `ref:<name>` |

### Email section

| Field | Required | Description |
|-------|----------|-------------|
| `template` | Yes | `ref:<template_name>` — must exist in workspace |
| `subject` | Yes | Email subject line (supports `{{profile.field}}`) |
| `html_file` | Yes | Path to HTML file (relative to YAML file's directory). For new campaigns, use the same HTML file used to create the template. For pulled campaigns, auto-generated by `tdx engage campaign pull`. |
| `variables` | No | Template variable definitions |

### Push section

| Field | Required | Description |
|-------|----------|-------------|
| `title` | Yes | Push notification title |
| `message` | Yes | Push notification body |
| `push_type` | Yes | `notification` |
| `on_click_behavior` | No | `open_deeplink` |
| `image_url` | No | Image URL for rich push |

## Segment References

The `ref:` prefix is **required** for `template`, `audience`, and `segment` fields. Raw IDs or plain names are rejected by the validator. Use `ref:` to reference by name:

```yaml
# Simple name reference (most common)
segment: ref:Loyalty Program Email Recipients

# With folder path (when name alone causes NOT_FOUND)
segment: ref:[1] Segments/Behavioral/Loyalty Program Email Recipients
```

The audience field always references a parent segment:
```yaml
audience: ref:engage_retail
```

## UTM Tracking

```yaml
utm:
  enabled: true
  id: 019xxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx  # Auto-assigned on pull
  medium: email                    # email | push
  source: treasuredata
  source_platform: treasuredata
  campaign: campaign-slug          # URL-safe campaign identifier
  marketing_tactic: tactic-name    # e.g., recommendations, spring-promo
```

The `id` field is auto-generated by the server. Omit when creating new campaigns.

## Connector Config

```yaml
connector:
  email_sender_id: 0199xxxx-xxxx  # Required for email campaigns
  all_columns: false               # true to send all columns
  columns:                         # Explicit column mapping
    - column: email
      source: email
    - column: first_name
      source: first_name
```

The `email_sender_id` is workspace-specific. Find it with `tdx delivery senders --workspace "Name"`.

## Variables

Email template variables for personalization:

```yaml
variables:
  - name: first_name         # Variable name used in template
    display_name: First Name  # Human-readable label
    preview_value: John       # Value shown in preview
    default_value: Customer   # Fallback when field is empty
```

Variables are referenced in HTML/subject as `{{profile.<name>}}`.
