# mail> Operator with HTML Templates

How to send HTML emails from TD Workflow using `mail>` with template files.

---

## Basic structure

1. Create an HTML file in `templates/` directory
2. Use `${variable}` syntax for digdag variable expansion
3. Reference it from `.dig` with `html_body:` parameter

## HTML template example

`templates/report.html`:

```html
<!DOCTYPE html>
<html>
<head>
<style>
  body { font-family: sans-serif; color: #333; line-height: 1.6; max-width: 700px; margin: 0 auto; padding: 20px; }
  .header { background: #1a1a2e; color: white; padding: 24px; border-radius: 8px; margin-bottom: 24px; }
  .header h1 { margin: 0; font-size: 22px; }
  .kpi-grid { display: grid; grid-template-columns: repeat(3, 1fr); gap: 12px; margin-bottom: 24px; }
  .kpi { background: #f8f9fa; border-radius: 8px; padding: 16px; text-align: center; }
  .kpi .value { font-size: 24px; font-weight: 700; }
  .kpi .label { font-size: 12px; color: #666; }
  .footer { margin-top: 24px; font-size: 12px; color: #999; text-align: center; }
</style>
</head>
<body>
  <div class="header">
    <h1>Daily Report</h1>
    <p>${session_date}</p>
  </div>

  <div class="kpi-grid">
    <div class="kpi">
      <div class="value">${td.last_results.metric_a}</div>
      <div class="label">Metric A</div>
    </div>
    <div class="kpi">
      <div class="value">${td.last_results.metric_b}</div>
      <div class="label">Metric B</div>
    </div>
    <div class="kpi">
      <div class="value">${td.last_results.metric_c}%</div>
      <div class="label">Metric C</div>
    </div>
  </div>

  <div class="analysis">
    ${analysis_content}
  </div>

  <div class="footer">
    Generated by TD Workflow
  </div>
</body>
</html>
```

## .dig file usage

```yaml
+send_report:
  mail>: templates/report.html
  subject: "Daily Report ${session_date}"
  to:
    - recipient@example.com
  html: true
```

## Available variables in templates

| Variable | Source | Example |
|----------|--------|---------|
| `${session_date}` | Workflow session | `2026-04-07` |
| `${session_time}` | Workflow session | `2026-04-07T00:00:00+00:00` |
| `${td.last_results.COLUMN}` | `store_last_results: true` on a prior `td>` task | KPI values |
| `${custom_var}` | `_export:` or `py>` with `digdag.env.store()` | Any custom value |

## Passing dynamic content (e.g. LLM output)

Use `py>` to store a value, then reference it in the template:

```python
# tasks/__init__.py
import digdag
import json

class ResponseExtractor:
    def run(self):
        raw = digdag.env.params["http.last_content"]
        data = json.loads(raw)
        text = data["content"][0]["text"]
        digdag.env.store({"llm_analysis": text})
```

```yaml
+extract:
  py>: tasks.ResponseExtractor.run

+send:
  mail>: templates/report.html
  subject: "Report ${session_date}"
  to:
    - recipient@example.com
  html: true
```

Template uses `${llm_analysis}` to embed the extracted text.

## Tips

- **Inline CSS only**: Email clients strip `<link>` tags. Keep all styles in `<style>` or inline `style=""` attributes.
- **No external images**: Some clients block external images. Embed key content as text/HTML, not images.
- **Max width ~700px**: Email clients have narrow viewports. Design for 600-700px width.
- **Test with plain text fallback**: The `mail>` body file itself is the HTML body when `html: true` is set. No separate plain text is sent.
